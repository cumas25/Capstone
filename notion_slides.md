# 1. 분석 배경 및 필요성

## 슬라이드 1 — 한국 배달로봇 시장, 지금 어디까지 왔나

### 법적 기반 마련 — 전국 보도 운행 허가 (2024)

- 2024년 「지능형 로봇 개발 및 보급 촉진법」 개정으로 **운행안전인증** 취득 시 전국 보도 통행 가능
- 인증 기준: 최대속도 **15km/h 이하**, 질량 **500kg 이하**, 경사도 대응 등 8개 항목 심사
- 법적 장벽은 해소됐지만, **실제 운영은 아직 일부 지역에 한정**

---

### 국내 주요 운영 기업

| 기업 | 로봇명 | 주요 운영 지역 | 운영 방식 |
|---|---|---|---|
| **뉴빌리티** | 뉴비 (Neubie) | 인천 송도, 판교, 서울 강남·서초 | 요기요 협력 공도 배달, 산업단지 식음료 배달, 아파트 도어투도어 |
| **우아한형제들 (배민)** | 딜리 (Dilly) | 서울 강남구 논현동·역삼동 | 배민B마트 시범 운영 (2025년 8월 차세대 모델 투입 예정) |

> 📌 두 기업 모두 운행안전인증 취득 완료 — 현재 **서울 강남권 집중**, 전국 확산은 진행 중

---

### 시장 전망

- 국내 배달로봇 시장 연평균 성장률 **약 40%** (한국과학기술정보연구원)
- 2021년 271만 달러 → **2026년 약 1,465만 달러** 규모로 성장 전망
- 글로벌 시장: 2025년 5.2억 달러 → 2034년 **76억 달러** (CAGR 35%)

---

> 💡 **현재 상황 요약**
> 법적 허가는 전국 완료 → 실제 운영은 강남권 등 일부 지역에 집중 → **데이터 기반으로 다음 확장 지역을 찾는 것이 본 분석의 출발점**

## 슬라이드 2 — 배달 산업의 구조적 한계와 서비스 공백

### 배달 산업의 구조적 한계
- 플랫폼 간 경쟁 심화로 **라이더 수급난 · 인건비 상승** 지속
- 소비자 배달비 부담 가중 + 플랫폼 수익성 악화의 악순환
- 비용 구조 개선을 위한 **대안 물류 수단 도입 시급**

### 배달로봇이 단거리 배달의 경제 구조를 바꾼다
- 라이더의 단거리 기피 원인은 물리적 장애가 아닌 **경제적 이유** — 건당 수입이 낮아서 회피
- 배달로봇은 **인건비가 없기 때문에** 단거리 배달도 수익이 나는 구조
- 즉, 로봇은 기피 구간을 "해소"하는 게 아니라 **단거리 배달의 수익 구조 자체를 전환**하는 수단
- → 인력이 구조적으로 커버하기 어려운 단거리 · 소량 배달을 **지속 가능한 형태로 서비스 가능**

---

## 슬라이드 3 — 왜 인프라 분석이 먼저인가 · 왜 서울인가

### 로봇 하드웨어 제약이 입지 선정의 선결 조건
- 현 상용 배달로봇 등판 한계: **경사도 8% 이하** / 권장 기준: **5% 이하**
- 배터리 효율 저하 · 모터 과부하가 주요 원인
- **수요가 아무리 높아도, 로봇이 물리적으로 주행 불가능한 지역은 도입 불가**
- → 인프라 가능 여부를 먼저 필터링한 뒤, 그 안에서 수요 우선순위를 정하는 것이 합리적

> 💡 **분석 설계 논리**
> 인프라(지형 · 보행 환경) → 1차 필터링 → 수요 밀도로 2차 우선순위 → 최적 지역 도출

### 서울, 최적 분석 도시로서의 조건
- 세계 최고 수준의 배달 밀도 (배달앱 이용률 · 음식점 수 기준)
- 평지(강남 · 마포) · 구릉지(관악 · 도봉)가 공존하는 **지형 다양성** → 경사도 분석의 변별력 확보
- 이미 일부 지역(강남구 논현 · 역삼동)에서 실증 운영 중 → **모델 검증 가능한 실제 데이터 존재**
- 전국 확산 전 테스트베드로 가장 적합한 도시

### 데이터 기반 전략적 확산 모델 구축
- 현재 운영은 특정 지역 시범 운영에 한정 → **서울 전역 확산 로드맵 부재**
- 배달 수요 + 도시 물리 환경 데이터를 결합한 **다차원 정량 분석** 필요
- 본 분석은 재현 가능한 프레임워크를 제시해 **타 도시 확장의 기반** 마련

# 2. 분석 목표 및 기대효과

## 슬라이드 4 — 분석 목표 및 기대효과

### 분석 목표

**목표 1. 데이터 기반 입지 우선순위 도출**
- 서울시 전체를 **500m × 500m 격자 단위**로 분할
- 인프라(지형 · 보행 환경) + 수요(음식점 밀도) 데이터 결합
- → 격자별 배달로봇 도입 **적합도 지수 산출 및 우선순위 지도 제작**

**목표 2. 타 도시로 확장 가능한 정량적 방법론 제시**
- GIS 공간 분석 + 다기준 점수화(MCS) 표준 파이프라인 구축
- → 서울 외 도시에 동일 방법론 적용 가능 / 데이터 갱신 시 자동 업데이트 가능한 구조

> 📌 **분석 범위 (Scope)**
> - 대상 지역: 서울시 전체 행정 구역
> - 분석 단위: 500m × 500m 격자
> - 수요 데이터: 실시간 주문량 미공개 → **음식점 · 편의점 밀도**로 대체
> - 실시간 보행자 유동인구: 유료 데이터 미사용 → **서울시 공개 집계 데이터** 대체

### 기대효과

| 대상 | 기대효과 |
|---|---|
| 🏢 민간 (배달 플랫폼) | 로봇 배달 시범 지역 확대 시 우선순위 근거 제공 |
| 🏛️ 공공 (서울시) | 스마트 모빌리티 정책 수립 · 보도 정비 우선순위 판단 지원 |
| 📚 학술 | 도시 물리 환경 × 배달 수요 결합 입지 분석 방법론 제시 |

# 3. 가설

## 슬라이드 5 — 가설 H1 ~ H4

### 핵심 가설

**H1 — 지형 조건 (가중치 35%)**
> 경사도 5% 이하 보행 구간 비율이 높은 격자일수록 배달로봇 운행 적합도가 높다.

- 표고점 KNN 기반 경사도 계산 → 최적(≤5%) · 허용(5~8%) · 불가(>8%) 분류
- 격자별 최적 구간 비율 산출 후 점수화

---

**H2 — 보행 네트워크 (가중치 25%)**
> 교차로 밀도가 낮고 링크 연결성이 높은 격자는 로봇 경로 계획이 단순해 운행 효율이 높다.

- 도보 네트워크 노드 수(교차로 밀도) · 링크 연결성 지수 산출
- 복잡도가 낮은 격자 = 로봇 경로 최적화에 유리

---

**H3 — 횡단 환경 (가중치 20%)**
> 횡단보도 · 신호등 밀도가 적정 수준인 격자는 로봇-보행자 충돌 리스크가 낮고 이동 예측 가능성이 높다.

- 횡단보도 위치 · 보행자 신호등 분포 격자 집계
- 과밀 지역 = 대기 시간 병목 발생 가능성 ↑

---

**H4 — 배달 수요 (가중치 20%)**
> 음식점 · 편의점 밀도가 높은 격자는 단거리 배달 수요가 집중되어 로봇 배달 운영 효율(회전율)이 높다.

- 행안부 인허가 API → 업종별 가맹점 수 격자 집계
- 수요 밀도 높은 격자에서 로봇 1대당 배달 건수 극대화

---

> 📌 4개 가설은 각각 독립 검증 후, **Spearman 상관분석**으로 최종 적합도 점수와의 관계 확인

## 슬라이드 7 — 수집 방법 및 환경

### 데이터 수집 방법

- **GIS 공간 데이터 (shp)**: 서울 열린데이터광장 직접 다운로드
- **인허가 데이터**: 행안부 localdata.go.kr REST API 호출
- **좌표계**: 모든 데이터 **EPSG:5179** (한국 중부원점 투영좌표계)로 통일

### 분석 환경

```
Python 라이브러리
- 공간 분석:  geopandas, shapely, pyproj
- 수치 계산:  numpy, scipy (cKDTree)
- 데이터 정제: pandas
- 시각화:     folium, matplotlib, seaborn
- API 수집:   requests
```

### 전처리 원칙
- 500m × 500m 격자 생성 후 **모든 데이터를 격자에 공간 조인(spatial join)**
- 결측 격자: 인근 격자 평균값 대체
- 이상치: IQR 기준 탐지 및 처리
- 각 지표 **Min-Max 정규화 (0~1)**

## 슬라이드 9 — STEP 2: 경사도 분석 (핵심 필터)

### 분석 방법

1. 표고점 데이터 로드 (shp → geopandas)
2. 좌표계 EPSG:5179 통일
3. **KNN(k=2)** 적용 → 가장 가까운 이웃 표고점 탐색
4. 경사도 = `|높이 차| / 수평 거리 × 100 (%)`
5. 이상값 제거 (100% 초과 제외)
6. 격자(500m)별 최적 비율 집계
7. Min-Max 정규화 → 경사 점수 산출

### 3단계 분류 기준

| 등급 | 경사도 기준 | 의미 |
|---|---|---|
| ✅ 최적 | 5% 이하 | 배달로봇 안정 운행 가능 |
| 🟡 허용 | 5 ~ 8% | 운행 가능하나 배터리 효율 저하 |
| ❌ 불가 | 8% 초과 | 등판 한계 초과, 운행 불가 |

### 한계 및 보완

- 직선 거리 계산 → 실제 보행 우회 경로 미반영 → **도보 네트워크 링크 기반 거리로 보정**
- 표고점 위치 자동 판별 불가(보도/건물 옥상 구분) → **도보 네트워크와 공간 조인으로 보도 위 점만 필터링**
- 표고점 간격 넓은 구간 급경사 미탐지 가능 → **네트워크 링크로 보완**

---

## 슬라이드 10 — STEP 4: 다기준 점수화 (MCS)

### 점수 산출 공식

> **격자 적합도 점수 = 지형(35%) + 보행 네트워크(25%) + 횡단 환경(20%) + 배달 수요(20%)**

### 가중치 테이블

| 영역 | 구성 지표 | 산출 방식 | 가중치 |
|---|---|---|---|
| 지형 조건 | 경사도 최적 구간 비율 | 표고점 KNN → 격자별 최적 비율 | **35%** |
| 보행 네트워크 | 교차로 밀도(역), 링크 연결성 | 도보 네트워크 노드/링크 공간 집계 | **25%** |
| 횡단 환경 | 횡단보도 · 신호등 밀도, 출입구 수 | 위치 데이터 격자 집계 → 부하 지수 | **20%** |
| 배달 수요 | 음식점 · 편의점 밀도 | 인허가 API → 지오코딩 → 격자 집계 | **20%** |

> ⚠️ (역) 표시 지표는 값이 클수록 불리 → **역방향 정규화(1 − 정규값)** 적용
> 가중치는 초안값이며, **민감도 분석(±10%p)** 후 최종 확정

---

## 슬라이드 11 — STEP 5~7: 군집화 · 검증 · 시각화

### STEP 5: 클러스터링
- **HDBSCAN** 또는 **K-Means** 적용 (최적 K: Elbow + 실루엣 점수)
- 군집 유형 해석 예: "고수요·저경사 최적", "고경사 운행 불가", "수요 부족 대기"
- 적합도 지수 상위 격자 → 구 단위 집계 → **우선 도입 후보 구 Top 5 선정**

### STEP 6: 검증
- 현재 실증 운행 지역(강남구 논현동 · 역삼동)과 모델 결과 대조 → **유효성 확인**
- 민감도 분석: 가중치 ±10%p 변경 시 순위 변동 폭 확인
- 후보 구 **로드뷰 육안 검증**: 공사 구간 · 좁은 골목 · 단차 구간 탈락 처리
- H1~H4 가설 Spearman 상관분석으로 검증

### STEP 7: 시각화
- 서울시 전체 격자 적합도 **히트맵** (Folium 인터랙티브 지도)
- 상위 후보 구 Top 5 **레이더 차트** (지형 / 네트워크 / 횡단 / 수요 점수)
- 군집별 특성 요약 (Kepler.gl)
